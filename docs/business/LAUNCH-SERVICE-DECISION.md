# Launch service decision

Prepared 2026-07-21 for owner sign-off. Reviews the 11 services (`SERVICE-OFFERING.md`, `data/services.ts`) against seven viability criteria: distinct customer, clear problem, defined deliverable, scope boundary, sensible entry price, recurring-support attach, appropriate conversion action. **No routes change** — this decision governs *sales emphasis and homepage/nav prominence*, not the URL register (all 11 service pages stay live and indexable; they earn search traffic regardless of emphasis).

## Classification

### Launch now (lead with these — promote on homepage, nav, proposals)

| Service | Why it passes all seven criteria |
|---|---|
| Business websites | Broadest demand ("website design/company" searches); clear deliverable (professional SEO-architecture site); entry via Starter/Professional packages; natural care-plan attach; CTA: request a quote. |
| Lead-generation websites | The positioning's core promise; distinct buyer (contractors/services who live on enquiries); measurable deliverable (enquiry tracking); Professional package entry; support attach high. |
| Product catalogue & RFQ websites | Flagship differentiator (segment 3 buyers); defined deliverable (catalogue + RFQ workflow); genuinely deliverable from founder's project history (A-13); premium price band; strong support attach. |
| Website redesign | High-intent buyer with an existing budget reference; clear problem (underperforming site); scope-boundable (audit → rebuild); redirects discipline is a credibility proof in itself. |
| Website maintenance & support | The recurring floor; attaches to every build; must exist from day one to catch the attach. |

### Secondary at launch (pages live and indexable; sold reactively, not led with)

| Service | Reason |
|---|---|
| SEO website development | Serves buyers who name SEO explicitly — keep the page for that intent, but selling "SEO" as a standalone invites ranking-guarantee expectations we refuse to set (rule 10). Positioned as "how we build", not a separate retainer. |
| Ecommerce websites | Real demand but heaviest builds; without a live ecommerce case study the proposal is weaker — quote selectively at launch. |

### Needs proof before promotion (sell via trust, promote after first case study)

| Service | Proof needed |
|---|---|
| Custom web applications | One permissioned system case study (property-management or admin/quotation history — owner input #7). |
| Admin panel development | Same evidence base; until then it's a conversation started from a website engagement. |
| RFQ & quotation systems | Strongest future differentiator; promote hard once a catalogue+RFQ build or the quotation-platform history is publishable. |
| Customer & supplier portals | Longest sales cycle; expansion offer for existing clients only at launch. |

### Could confuse the initial positioning (watch, don't cut)

- None require removal. The Tier-2 system services are the agreed expansion layer (D-06); the risk is only *emphasis* — if the homepage leads with "custom software", the D2 website buyer bounces. Mitigation: homepage leads with websites + lead generation; systems appear as the growth path ("the website, and the system behind it").

### Consolidation check

- No consolidation recommended. Each service page maps to a distinct documented search intent (`docs/seo/KEYWORD-TO-PAGE-MAP.csv`), and D-13 already prevents package/service cannibalisation. Support sub-offerings correctly live on one page (`SUPPORT-PACKAGES.md`) — keep that.

## Package structure (recommendation: keep all five, stagger emphasis)

| Package | Launch emphasis |
|---|---|
| Starter business website | Floor offer — quoted, not promoted. Exists to catch smaller budgets without dragging the brand down-market. |
| Professional business website | **Hero package** — the default proposal for segments 1, 2, 5. |
| Product catalogue website | **Flagship** — the differentiation showcase for segment 3. |
| Ecommerce website | Live, quoted on request at launch. |
| Custom web system | Discovery-first entry (paid discovery R8k–R15k indicative) — protects against unscoped custom work. |

## Conversion actions (confirm)

- Build services/packages → "Request a quote" (form).
- Systems services → "Book a consultation" (lower commitment fits longer cycle).
- Maintenance → "Request a quote" with plan pre-selected.
