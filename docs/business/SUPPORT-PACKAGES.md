# Support packages (recurring services)

All amounts are **indicative placeholders** — confirm before publishing (owner input #6). Sold monthly, cancel with 30 days’ notice (proposed term — confirm). One public page (`/services/website-maintenance-and-support/`) presents all plans to avoid thin sibling pages.

| Plan | For | Core inclusions | Indicative /month |
|---|---|---|---|
| Hosting administration | Any site we host/manage | Hosting + domain admin, SSL, DNS, uptime monitoring, backups | R450–R850 |

| Web application support | Custom systems | SLA response targets, bug fixes, dependency updates, small enhancements backlog | R3,500–R8,000 |
| Search performance reporting | Any client | Monthly GSC/GA report with plain-language actions | R950–R1,800 (add-on) |
| Content and SEO support | Growth clients | 1–2 mapped articles/pages per month + internal linking + metadata upkeep | R4,500–R9,000 |

## Rules

1. Support is attached at proposal stage of every build (“every build quote includes a recommended care plan”).
2. Plans are deliverable-defined, not hour-defined where possible; unused hours don’t roll over (proposed — confirm).
3. No plan promises rankings; reporting shows real GSC data only.
4. Out-of-plan work billed at a published hourly rate (indicative R650–R950/h — market-aligned; confirm).

## Revenue logic

Ten Business-support clients ≈ R18k–R35k/month recurring — the cash-flow floor that de-risks project gaps. Ecommerce/application support carries higher rates because downtime = client revenue loss.
